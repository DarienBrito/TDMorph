TDMorph

version 3.0

About

I made TDMorph to enhance parametric exploration, preset storage, composition, and cueing in the TouchDesigner platform. It helps create generative content via various methods and archiving interesting parametric configurations. Furthermore, the library provides a set of tools that developers can use to create their custom systems. The project showcases a possible way of working when creating and exploring computer graphics systems.

The inspiration comes from my background in algorithmic music composition, taking from ideas found in Gottfried Michael Koenig's Project 1 and Project 2, Paul Berg's AC Toolbox, Alberto di Campo's CloudGenMini and my own NLC toolbox, in the SuperCollider programming language.

What can I do with it?

  • A random search of states for parameters on any node or number of nodes.
  • Random morphing between states using various interpolation curves.
  • Storing and retrieval of any arbitrary number of presets.
  • Control of parameters and transitions with automatically generated UIs.
  • Automatic storing, randomization and transformation of all parameters of arbitrary nodes.
  • Auto-learn MIDI and OSC for every control in the generated UI.
  • Scripting control of all features via high-level commands.
  • Global and local timing controls for every single parameter.
  • Automatic creation of animations from stored presets.
  • Flexible cueing system with follow actions and quantization.
  • Pattern generation via scripting with the Patterns library.
  • Algorithmic cueing via scripts.
And much more!

How does it work?

The architecture of TDMorph is modular, allowing you to access the core functionality as a whole or as separate building blocks. The package consists of three different objects:

  • The PresetManager node. A powerful object to control any arbitrary number of nodes and parameters at once. You can manage their states with a minimal UI or directly with code. The PresetManager is the backbone of the TDMorph engine, suitable for advanced developers who want to make custom systems.
  • The ParametersMorpher node. A high-level UI where you can drag and drop parameters from any node and create controls automatically, along with a large set of controls for presets and transformations.
  • The SceneLauncher node. A high-level UI where you can connect any PresetManager and control scenes and presets similar to cue systems like CueLab or the Scene Launcher system in Ableton Live.

Motivation

The motivation to share this tool stems from the wonderful sense of comradery that the TouchDesigner community in general has and by the brilliant philosophy of Derivative https://derivative.ca/, as creators of the TouchDesigner software. I hope that the ethos that characterizes TD continues and that this system helps you in expanding your capabilities as a maker.
